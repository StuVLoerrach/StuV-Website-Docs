# Feature Section

The Feature Section component showcases key features, benefits, or services offered.

## Overview

Feature sections help highlight the main capabilities, benefits, or services of your product or organization. They are designed to communicate value propositions clearly and effectively.

## Props

| Prop | Type | Description | Default |
|------|------|-------------|---------|
| `variant` | `string` | The variant of the feature section to display | `"variant1"` |
| `title` | `string` | The main heading text | `""` |
| `subtitle` | `string` | The secondary heading text | `""` |
| `description` | `string` | The descriptive text | `""` |
| `features` | `Feature[]` | Array of features to display | `[]` |
| `backgroundColor` | `string` | Background color of the section | `"bg-white"` |
| `textColor` | `string` | Text color of the section | `"text-gray-900"` |
| `layout` | `"grid" \| "list" \| "alternating"` | Layout style | `"grid"` |
| `columns` | `number` | Number of columns in grid layout | `3` |
| `alignment` | `"left" \| "center" \| "right"` | Text alignment | `"center"` |
| `iconStyle` | `"outline" \| "solid" \| "duotone" \| "none"` | Style of icons | `"outline"` |
| `iconColor` | `string` | Color of icons | `"text-primary"` |

## Accessibility

- Icons have appropriate aria-hidden attributes
- Proper heading hierarchy is maintained
- Interactive elements are keyboard accessible
- Color contrast meets WCAG AA standards
- Tab interfaces follow WAI-ARIA best practices
